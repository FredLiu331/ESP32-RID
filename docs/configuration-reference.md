# 配置参考

## 默认值和限制

| 项目 | 默认值/限制 |
|---|---|
| 中心点 | 无；应用前必须设置 |
| 全局消息周期 | 1000 ms |
| 周期范围 | 100..60000 ms |
| 机群总数 | 1..50 |
| Wi-Fi 封装 | Beacon |
| Wi-Fi 2.4 GHz | 信道 6 |
| Wi-Fi 5.8 GHz | 信道 149 |
| NAN | 禁止 |

配置按“全局 -> 组 -> 运行实例”展开。运行实例 ID 由设备身份、组名、组内索引和协议
确定性派生；同一设备上保持组配置不变时，重启后 ID 不变。用户不需要也不能导入真实无人机序号。

## 支持组合

- `gb` 只能配 Wi-Fi Beacon（`wifi24` 或 `wifi5/wifi58`）。
- `odid` 可配 `ble4`、`ble5`、`wifi24` 或 `wifi5/wifi58`。
- 所有组都使用 Beacon 语义；不接受 NAN。
- 轨迹模型由固件默认配置生成，当前 Shell 尚未开放轨迹参数修改。

## 事务顺序

1. 用 `site set` 和 `group add` 修改暂存配置。
2. 用 `config check` 校验中心点、总架次、组名、周期和承载组合。
3. 用 `config apply` 生成运行实例并开始持续发送，代次加一。
4. 用 `config save` 将已应用快照写入 NVS 双槽存储。
5. 下次上电时加载 CRC 正确且代次最新的快照；无有效快照则保持停发并等待 Shell。
